# 开船第一课优化方案

## 🎯 核心问题分析

### AI生成的问题：
1. **主题错位**：把"香港保险配置"改成了"香港储蓄分红险"
2. **内容空泛**：大而空的"写给不确定时代"，缺乏实操性
3. **结构混乱**：包含了不需要的***等格式符号
4. **缺少关键要素**：没有航线图、开船工具等实操指引

### 手工编写的优点：
1. **主题准确**：严格按"香港保险配置"
2. **内容具体**：直接的项目介绍，用户画像清晰
3. **结构清晰**：1.1、1.2、1.3...的清晰编号
4. **实操性强**：有具体的操作路径和工具清单

## 🔧 修改策略（删除对齐，不是增加）

### 1. 删除AI自作主张的部分
- ❌ 删除"理解项目 vs 工具"的复杂逻辑（6,826字→约1,500字）
- ❌ 删除过度详细的格式要求
- ❌ 删除"写给不确定时代"这种包装性语言

### 2. 强化主题对齐约束
- ✅ 明确："项目名称必须一字不改"
- ✅ 明确："禁止AI理解或修改主题"
- ✅ 明确："直接用手册标题，不是AI理解"

### 3. 简化内容要求
- ✅ 从"大而全"→"具体实操"
- ✅ 从"AI理解"→"手册提取"
- ✅ 从"包装性语言"→"直接说是什么"

## 📝 新提示词框架

```
你是一位专业的航海训练营内容策划师。请根据提供的航海手册内容和详情页内容，生成"开船第一课"。

【核心约束 - 必须遵守】
1. **项目名称必须一字不改**：使用手册封面/标题中的准确名称，禁止AI理解或修改
2. **内容必须来自手册**：所有案例、数据、方法必须100%从手册提取，严禁编造
3. **不要大而空**：用具体、实操的内容，不是泛泛而谈

【内容要求】
1. 开篇直接说项目是什么，怎么玩（基于手册第一页）
2. 用户画像要具体：从手册中提取明确的适用人群
3. 操作路径要落地：从详情页复制"项目关键步骤"，一字不改
4. 避坑指南要实用：从手册提取真实风险和解决方案

【格式要求】
- 大标题：⛵新手如何入门【项目名称】项目？
- 优势用✅，鼓励用🎗️
- 不使用##标题符号
- 只在操作路径中使用**加粗步骤**
```

## 🔍 关键检查点

生成后必须检查：
1. ✅ 项目名称是否和手册标题完全一致
2. ✅ 是否删除了"写给不确定时代"这类空话
3. ✅ 用户画像是否具体明确（不是泛泛的"创业者"）
4. ✅ 是否包含了"项目关键步骤"和"开船前必备工具"
5. ✅ 格式是否简洁，没有***等冗余符号
