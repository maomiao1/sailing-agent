# 开船第一课微调方案

## 🚨 核心问题识别

### 1. 主题错位问题（最严重）
**AI生成**：把"香港保险配置"→"香港储蓄分红险"
**手工版**：保持"香港保险配置"
**微调点**：在现有提示词中加强"主题必须一字不改"约束

### 2. 内容空泛问题
**AI生成**："写给在不确定时代寻求确定性的你"
**手工版**："1.1 为什么做这个航海？"直接具体
**微调点**：在"内容要求"部分增加"避免空洞包装语言"

### 3. 内容编造问题
**AI生成**：生成了手册没有的大而空内容
**手工版**：每个点都有具体数据/案例支撑
**微调点**：强化"必须基于手册内容，禁止编造"约束

## 🔧 微调整改方案

### **修改1：在项目识别部分（约第30行）**

**原文**：
```
3. 极其重要：准确识别当前主项目名称（从手册封面、标题、第一页、目录），不要被手册中间部分的旧项目内容或工具名称误导
```

**微调为**：
```
3. 极其重要：准确识别当前主项目名称 - 必须使用手册封面/标题中的准确名称，一字不改，禁止AI理解或修改
4. **禁止编造**：不要在项目名称前加形容词、不要修改主题方向、不要替换成AI理解的概念
```

### **修改2：在内容要求部分（约第100行）**

**增加约束**：
```
【内容真实性要求】
- 所有内容必须100%基于当前上传的航海手册，严禁编造任何手册中没有的内容
- 如果发现手册中没有某个信息，不要编造，直接跳过该部分
- 用户画像、案例、数据必须来自手册原文，不是AI的理解
```

### **修改3：在格式/风格部分**

**增加**：
```
【内容风格要求】
- 避免"写给不确定时代"这类空洞包装语言
- 直接说"这是什么"而不是"本项目旨在"
- 用具体数据/案例说话，不是泛泛而谈
```

## ✅ 保留的关键要素

1. **完整框架**：保持原有6步生成流程
2. **格式规则**：保持对**符号、emoji的精确要求
3. **引用机制**：保持"从详情页复制关键步骤"机制
4. **项目识别**：保持多步骤验证机制

1. 在关键位置增加"禁止AI自作主张"约束
